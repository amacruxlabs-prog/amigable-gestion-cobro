Autenticación

Todos los endpoints (excepto /public/businesses) requieren autenticación mediante dos credenciales:

X-Entity-ID

ID de la entidad o agente autorizado. Se obtiene al registrar tu agente en el panel de administración.

X-Entity-ID: 779d1ab2-2bc1-4f27-b6a8-9e99cc00cd64
Authorization: Bearer

Token de acceso. Se genera desde el panel SuperAdmin en /superadmin/api-entities. También puede enviarse como X-API-Key.

Authorization: Bearer 81381ab9d12898b9137b40590b4dfb884f420fbee01a5f102961584396b0fb99
Importante: Ambas credenciales deben enviarse en el header HTTP de cada petición. El token puede expirar si se configuró una fecha de expiración al generarlo.

Ejemplo con cURL

curl -sk "https://api-amigablecobro.amacruxlab.com/api/v1/collections?business_uuid=b206bc66-758b-4c8a-87e0-cf85395e57d7" \
  -H "X-Entity-ID: 779d1ab2-2bc1-4f27-b6a8-9e99cc00cd64" \
  -H "Authorization: Bearer 81381ab9d12898b9137b40590b4dfb884f420fbee01a5f102961584396b0fb99" \
  -H "Accept: application/json"

POST /collections

Crea una nueva cuenta por cobrar.

Body (JSON)

CampoTipoDescripción
business_uuidstringreqUUID del negocio (ej: b206bc66-...
client_namestringreqNombre del cliente/deudor
client_phonestringoptTeléfono (ej: +584241234567)
client_documentstringoptCédula o RIF
total_amountfloatreqMonto total de la deuda
due_datestringoptFecha de vencimiento (ISO 8601)
pageintegeroptNúmero de página (default: 1)

Ejemplo

curl -sk -X POST https://api-amigablecobro.amacruxlab.com/api/v1/collections \
  -H "X-Entity-ID: 779d1ab2-..." \
  -H "Authorization: Bearer 81381ab9..." \
  -H "Content-Type: application/json" \
  -d '{
    "business_uuid": "b206bc66-758b-4c8a-87e0-cf85395e57d7",
    "client_name": "Juan Pérez",
    "client_phone": "+584140000000",
    "client_document": "V12345678",
    "total_amount": 250.00,
    "due_date": "2026-08-15T00:00:00Z"
  }'

Respuesta (201 Created)

{
  "success": true,
  "data": {
    "id": 10,
    "client_name": "Juan Pérez",
    "client_phone": "+584140000000",
    "client_document": "V12345678",
    "total_amount": 250.00,
    "paid_amount": 0.00,
    "status": "PENDING",
    "due_date": "2026-08-15T00:00:00Z",
    "created_at": "2026-06-24T12:00:00Z"
  }
}

Parámetros

ParámetroUbicaciónDescripción

business_uuidqueryreqID del negocio

  -H "X-Entity-ID: 779d1ab2-..." \
  -H "Authorization: Bearer 81381ab9..."
POST /collections/{id}/payments

Registra un pago parcial o total a una cuenta.

Body (JSON)

CampoTipoDescripción
business_uuidstringreqUUID del negocio (ej: b206bc66-...
amountfloatreqMonto del pago
payment_methodstringreqMétodo: CASH, TRANSFER, POS, ZELLE, PAYPAL, OTHER
payment_datestringoptFecha del pago (ISO 8601). Default: ahora

Ejemplo

curl -sk -X POST https://api-amigablecobro.amacruxlab.com/api/v1/collections/10/payments \
  -H "X-Entity-ID: 779d1ab2-..." \
  -H "Authorization: Bearer 81381ab9..." \
  -H "Content-Type: application/json" \
  -d '{
    "business_uuid": "b206bc66-758b-4c8a-87e0-cf85395e57d7",
    "amount": 100.00,
    "payment_method": "TRANSFER",
    "payment_date": "2026-06-24T14:30:00Z"
  }'

Códigos de Error

HTTPCódigoSignificado
401Faltan credencialesNo se envió X-Entity-ID o el token Bearer
401Credenciales inválidasEl token o entity ID no coinciden
401Token expiradoEl token tiene una fecha de expiración vencida
403Entidad inactivaLa entidad API fue desactivada desde el panel
404No encontradoEl recurso solicitado no existe
422Error de validaciónDatos inválidos en la petición (revisar campos)
500Error internoError del servidor. Contactar al administrador.
Todos los errores retornan un JSON con la estructura: { "success": false, "message": "descripción del error" }
