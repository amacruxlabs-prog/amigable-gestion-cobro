Listo para probar
Haz clic en cualquier endpoint para expandirlo. Verás la descripción, los parámetros que recibe, la estructura de la respuesta, y podrás probarlo con datos reales usando el botón Probar. Las credenciales se envían automáticamente. También puedes copiar el comando cURL equivalente.

Autenticación

Todos los endpoints requieren autenticación excepto /public/businesses. Debes enviar dos headers en cada petición.

Header
X-Entity-ID
Valor
UUID v4
Descripción
Identificador único de la entidad o agente autorizado. Se obtiene al registrar un agente en el panel SuperAdmin (/superadmin/api-entities). Cada agente tiene su propio UUID que lo identifica ante el sistema.
Header
Authorization
Valor
Bearer {token}
Descripción
Token de acceso generado desde el panel SuperAdmin. El token puede tener fecha de expiración. También puede enviarse como X-API-Key como alternativa. Si el token expira, deberás generar uno nuevo desde el panel. 
curl -sk "https://api-amigablecobro.amacruxlab.com/api/v1/collections?business_uuid=b206bc66-..." \
  -H "X-Entity-ID: 779d1ab2-..." \
  -H "Authorization: Bearer 81381ab9..." \
  -H "Accept: application/json"

Directorio Público 1

GET /public/businesses
Propósito

Obtiene el listado de todos los negocios activos registrados en la plataforma. Cada negocio incluye su UUID (necesario para el resto de endpoints), nombre comercial y teléfono de WhatsApp.

¿Para qué sirve?

Es el primer paso para cualquier integración. Llama a este endpoint para descubrir qué negocios están disponibles y obtener su uuid, que usarás como business_uuid en todas las demás peticiones.

Autenticación

No requiere — es un endpoint público.

Respuesta

CampoTipoDescripción
uuidstringUUID del negocio. Úsalo como business_uuid en los demás endpoints.
namestringNombre comercial del negocio (ej: "Club Árabe de Maturín").
whatsapp_phonestring|nullNúmero de teléfono de WhatsApp asociado al negocio.

Colecciones 5

GET /collections
Propósito

Lista todas las cuentas por cobrar (transacciones) de un negocio. Retorna resultados paginados (30 por página) ordenados por fecha de creación descendente (más recientes primero).

¿Cuándo usarlo?

Para obtener el listado completo de deudores de un negocio, consultar el estado de cada cuenta, y filtrar por estado de pago (PENDING o PAID).

Parámetros (query string)

ParámetroTipoDescripción
business_uuidstringreqUUID del negocio (obtenido de /public/businesses)
statusstringoptFiltrar por estado: PENDING (pendiente) o PAID (pagado)
start_datestringoptFecha inicial para filtrar por rango (ISO 8601, ej: 2026-06-01)
end_datestringoptFecha final para filtrar por rango (debe ser posterior a start_date)
pageintoptNúmero de página para paginación (empieza en 1). Por defecto: 1

Respuesta

{
  "success": true,
  "data": [
    {
      "id": 1,                    // ID interno de la transacción
      "client_name": "Juan Pérez", // Nombre del deudor
      "client_phone": "+584241234567", // Teléfono
      "client_document": "V12345678", // Cédula o RIF
      "total_amount": 250.00,      // Monto total de la deuda
      "paid_amount": 50.00,        // Monto pagado hasta ahora
      "status": "PENDING",         // Estado actual
      "due_date": "2026-08-15T00:00:00Z",  // Fecha de vencimiento
      "created_at": "2026-06-24T12:00:00Z" // Fecha de creación
    }
  ],
  "meta": {
    "total": 50,        // Total de transacciones
    "current_page": 1,  // Página actual
    "last_page": 5      // Última página disponible
  }
}
POST /collections
Propósito

Registra una nueva cuenta por cobrar en el sistema. Crea una transacción con estado inicial PENDING y monto pagado en cero.

¿Cuándo usarlo?

Cuando un negocio emite una nueva deuda a un cliente. Por ejemplo: una membresía, un servicio prestado a crédito, o una cuota pendiente. El sistema automáticamente asigna estado PENDING.

Cuerpo de la petición (JSON)

CampoTipoDescripción
business_uuidstringreqUUID del negocio al que pertenece la cuenta
client_namestringreqNombre completo del cliente o deudor
client_phonestringoptTeléfono del cliente (formato internacional: +584241234567)
client_documentstringoptCédula de identidad o RIF del cliente
total_amountfloatreqMonto total de la deuda en bolívares o dólares
due_datestringreqFecha de vencimiento en formato ISO 8601 (2026-08-15T00:00:00Z)

Respuesta (201 Created)

{
  "success": true,
  "message": "Cuenta creada",
  "data": {
    "id": 10  // ID de la nueva transacción creada
  }
}
GET /collections/{id}
Propósito

Obtiene el detalle completo de una cuenta por cobrar específica, incluyendo el historial de pagos registrados. Útil para conocer el estado actualizado de una deuda.

¿Cuándo usarlo?

Cuando necesitas consultar el estado detallado de una cuenta en particular: saber cuánto se ha pagado, qué método de pago se usó, y el historial completo de abonos.

ParámetroUbicaciónTipoDescripción
idURL (path)intreqID de la transacción a consultar
business_uuidquerystringreqUUID del negocio

POST /collections/{id}/payments
Propósito

Registra un pago parcial o total a una cuenta por cobrar existente. El sistema actualiza automáticamente el monto pagado y, si el total pagado alcanza o supera el monto total, cambia el estado a PAID.

¿Cuándo usarlo?

Cuando un cliente realiza un abono a su deuda. Puedes registrar pagos parciales (ej: abono de Bs. 50 en una deuda de Bs. 250) o pagos completos. El sistema lleva el control del saldo restante automáticamente.

ParámetroUbicaciónTipoDescripción
idURL (path)intreqID de la transacción a la que se aplica el pago

Cuerpo de la petición (JSON)

CampoTipoDescripción
business_uuidstringreqUUID del negocio
amountfloatreqMonto del pago (debe ser mayor a 0)
payment_methodstringreqMétodo de pago: CASH, TRANSFER, POS, ZELLE, PAYPAL, OTHER
payment_datestringoptFecha del pago en ISO 8601. Si no se envía, se usa la fecha/hora actual.

Respuesta

{
  "success": true,
  "message": "Pago registrado",
  "data": {
    "new_paid_amount": 150.00,  // Nuevo total pagado acumulado
    "new_status": "PARTIAL"      // Estado actualizado de la cuenta
  }
}
// Posibles errores:
// 400 - "La cuenta ya está pagada" (status = PAID)
// 404 - "Transacción no encontrada"
PUT /collections/{id}/status
Propósito

Actualiza manualmente el estado de una cuenta por cobrar. Permite cambiar entre los estados disponibles sin necesidad de registrar un pago.

¿Cuándo usarlo?

Para gestionar el ciclo de vida de una deuda: marcar como PAID cuando se cobró en efectivo fuera del sistema, como OVERDUE cuando venció, o CANCELLED si se anula. También para revertir a PENDING si es necesario.

ParámetroUbicaciónTipoDescripción
idURL (path)intreqID de la transacción

Estados disponibles

ValorSignificado
PENDINGPendiente por cobrar (estado inicial)
PAIDPagada completamente
OVERDUEVencida / en mora
CANCELLEDCancelada (ya no es exigible)

Cuerpo de la petición (JSON)

CampoTipoDescripción
business_uuidstringreqUUID del negocio
statusstringreqNuevo estado: PENDING, PAID, OVERDUE, CANCELLED
{
  "success": true,
  "message": "Estado actualizado"
}

Estadísticas 1

GET /analytics/kpis
Propósito

Obtiene los indicadores clave de rendimiento (KPIs) de un negocio: total emitido, total cobrado, saldo pendiente, tasa de cobranza, y conteo de cuentas activas vs pagadas.

¿Cuándo usarlo?

Para dashboard y reportes. Útil para mostrar en tiempo real la salud financiera del negocio: cuánto se ha facturado, cuánto se ha cobrado, y la eficiencia de cobranza en un período específico.

Parámetros (query string)

ParámetroTipoDescripción
business_uuidstringreqUUID del negocio
periodstringoptPeríodo de análisis: today, this_week, this_month (default), this_year, all_time

Respuesta

CampoTipoDescripción
total_emittedfloatSuma total de montos emitidos (facturados)
total_collectedfloatSuma total de montos cobrados
total_pendingfloatMonto pendiente por cobrar (total_emitted - total_collected)
collection_ratefloatPorcentaje de eficiencia de cobranza (0-100)
active_accountsintCuentas activas con estado pendiente
paid_accountsintCuentas pagadas completamente

Calendario 1

GET /calendar/events
Propósito

Obtiene los eventos de cobro agrupados por día para un mes y año específicos. Cada día incluye el total esperado, total cobrado, y la lista de transacciones con sus montos y estados.

¿Cuándo usarlo?

Para visualizar en un calendario los vencimientos y cobros de cada día. Ideal para que el negocio sepa qué días tiene más cuentas por cobrar y planifique su gestión de cobranza.

ParámetroTipoDescripción
business_uuidstringreqUUID del negocio
monthintreqMes (1-12)
yearintreqAño (ej: 2026)

Respuesta

{
  "success": true,
  "data": [
    {
      "date": "2026-06-15",              // Día del evento
      "total_expected": 500.00,           // Total esperado para este día
      "total_collected": 200.00,          // Total cobrado hasta ahora
      "events": [                         // Transacciones de este día
        {
          "transaction_id": 5,
          "client_name": "María García",
          "amount": 500.00,
          "status": "PARTIAL"
        }
      ]
    }
  ]
}
// Si no hay eventos en un mes, retorna un array vacío.
